Для подготовки отчетных материалов по учебной практике необходимо строго следовать установленной структуре. Ниже приведены требования к наполнению основных файлов архива.

### Требования к структуре документации

1. README.md (Основное описание) 

Файл должен содержать:

* 
**Цель**: описание назначения датасета и задач машинного обучения (регрессия, классификация, кластеризация).


* 
**Предметная область**: описание объектов датасета.


* 
**Признаки**: таблица с указанием имени, типа, роли и описания каждого признака.


* 
**Целевая переменная**: явное указание `target`.


* 
**Особенности данных**: описание пропусков, выбросов и специфических типов данных (DateTime, категориальные и т.д.).


* 
**Использование**: инструкция по загрузке и запуску примеров.


* 
**Ограничения**: информация о синтетических частях, упрощениях и известных проблемах качества.



2. quality_report.md (Отчет о качестве) 

Файл должен включать:

* 
**Размер**: количество строк и столбцов.


* 
**Типы признаков**: классификация по типам (числовые, категориальные, строковые, булевы, DateTime).


* 
**Пропуски и выбросы**: указание долей пропусков и описание необычных значений.


* 
**Категориальные признаки**: число уникальных значений для каждого.


* 
**Рекомендованная обработка**: шаги по заполнению пропусков, кодированию признаков, обработке выбросов и извлечению признаков из DateTime.



3. ai_log.md (Лог использования ИИ) 

Обязательные разделы:

* 
**Инструменты**: список использованных ИИ (ChatGPT, GitHub Copilot, Cursor и др.).


* 
**Цели использования**: например, выбор темы, генерация данных, написание кода или подготовка мета-файлов.


* 
**Ручная проверка**: перечисление того, что было проверено лично: соответствие CSV и meta, типы признаков, корректность `target`, запуск примеров, отсутствие персональных данных и разумность значений.



4. sources.md (Источники данных) 

* Для **полностью синтетических** данных: указать, что они синтетические и персональные данные не использовались.


* Для **частично синтетических**: указать, что структура основана на сценариях предметной области.


* Для **реальных данных**: указать название источника, прямую ссылку, дату обращения, лицензию и выполненные преобразования.



---

Примечание: Директория `scripts/` (с файлами типа `preprocessing.py`) является обязательной только для больших реальных датасетов или случаев с нетривиальной обработкой данных.
