### 3. Спецификация признаков (Структура таблицы)

Студентам предоставляется плоская таблица со следующей архитектурой полей:

1. **`year`** (Integer): Год фиксации наблюдения (0 пропусков). Диапазон: 2007–2022.

2. **`region`** (String): Название субъекта Российской Федерации (0 пропусков).

3. **`region_oktmo`** (String): Код Общероссийского классификатора территорий муниципальных образований для региона (0 пропусков).

4. **`city`** (String): Наименование города или населенного пункта (0 пропусков).

5. **`city_oktmo`** (String): Код ОКТМО населенного пункта (30 пропусков).

6. **`air_general_level`** (Integer): Общий уровень загрязнения воздуха (473 пропуска).  
   Значения:
   - `1` — низкий (ИЗА 0–4, СИ < 5, НП < 20%)
   - `2` — повышенный (ИЗА 5–6, СИ < 5, НП < 20%)
   - `3` — высокий (ИЗА 7–13, СИ 5–10, НП 20–50%)
   - `4` — очень высокий (ИЗА ≥ 14, СИ > 10, НП > 50%)  
   *ИЗА — комплексный индекс загрязнения атмосферы; СИ — стандартный индекс (макс. разовая концентрация / ПДК); НП — наибольшая повторяемость превышения ПДК (%).*  
   **Методический капкан:** В некоторых городах уровень не оценён из-за недостаточного количества данных наблюдений или числа веществ, необходимых для расчёта ИЗА.

7. **`air_standard_index`** (String): Перечень веществ, для которых Стандартный Индекс (СИ) превысил 10 ПДК (3421 пропуск; `No substance` — отсутствие превышений).  
   Сокращения веществ:
   | Код | Вещество | Код | Вещество |
   |-----|----------|-----|----------|
   | БП | бенз(а)пирен | NO₂ | диоксид азота |
   | ВВ | взвешенные вещества (пыль) | NO | оксид азота |
   | Ф | формальдегид | NH₃ | аммиак |
   | ЭБ | этилбензол | СО | оксид углерода |
   | Тв.НF | твердые фториды | SO₂ | диоксид серы |
   | НF | фторид водорода | CS₂ | сероуглерод |
   | HCl | хлорид водорода | H₂S | сероводород |
   | РМ | взвешенные частицы (PM) | | |

8. **`air_repeatability`** (String): Наибольшая повторяемость (НП) превышения максимальной разовой ПДК в процентах с указанием вещества (3713 пропусков или `No substance`).  
   *НП — наибольшая повторяемость (в %) превышения макс. разовой ПДК по данным наблюдений за одним веществом на всех постах за месяц и/или год.*

9. **`air_qcp`** (String): Список веществ, у которых среднегодовая концентрация превысила 1 ПДК (1110 пропусков или `No substance`).  
   *ПДК — предельно допустимая концентрация для населенных мест, устанавливаемая Главным санитарным врачом РФ.*

10. **`air_solid_emissions`** (Float): Объем промышленных выбросов твердых веществ, тыс. тонн (471 пропуск, скрытый за `-1.0`).

11. **`air_so_emissions`** (Float): Объем промышленных выбросов диоксида серы SO₂, тыс. тонн (482 пропуска, скрытых за `-1.0`).

12. **`air_no_emissions`** (Float): Объем промышленных выбросов диоксида азота NO₂, тыс. тонн (451 пропуск, скрытый за `-1.0`).

13. **`air_co_emissions`** (Float): Объем промышленных выбросов монооксида углерода CO, тыс. тонн (450 пропусков, скрытых за `-1.0`).

14. **`air_population`** (Float): Численность населения города, тыс. человек (168 пропусков).

15. **`air_stantions`** (String): Количество действующих станций мониторинга в населенном пункте (1 пропуск или `No information`).  
    Формат: целое число (например, `1`), суммарное количество станций основной и ведомственной сети (например, `5+1` — 5 станций Росгидромета + 1 ведомственная), или `эп` — эпизодические наблюдения.

### 4. Требования к реализации ML-задач

#### Задача А. Регрессия (Прогнозирование объемов выбросов)

* **Целевая переменная:** `air_co_emissions` (или любой другой промышленный газ на выбор).

* **Суть задачи:** Построить модель, предсказывающую объем выбросов угарного газа (в тыс. тонн) на основе численности населения, года, региона и объема выбросов твердых фракций.

* **Метрики оценки:** $R^2$ и $MAE$.

* **Доступные модели в MLABC:** `KNNRegressor`.

* **Рекомендация:** Сравнить результаты при разных значениях $K$ (например, 3, 5, 7, 10). Использовать `StandardScaler` перед обучением, так как признаки имеют разные масштабы (тыс. тонн vs тыс. человек).

#### Задача Б. Мультиклассовая классификация (Оценка экологического риска)

* **Целевая переменная:** `air_general_level` (значения от 1 до 4).

* **Суть задачи:** Оценить итоговый уровень опасности для города (от низкого до очень высокого) на основе структуры промышленных выбросов.

* **Метрики оценки:** Поскольку классы несбалансированы, использовать взвешенную метрику $F_1\text{-score}$. Построить матрицу ошибок (вывести в виде таблицы через `Print`).

* **Доступные модели в MLABC:** `KNNClassifier`, `LogisticRegression`.

* **Рекомендация:** Для мультиклассовой задачи `LogisticRegression` может работать только в бинарном режиме — в этом случае использовать `KNNClassifier`. Удалить строки с пропущенным `air_general_level` (473 записи) перед обучением.

#### Задача В. Кластеризация (Типологизация промышленных зон)

* **Входные признаки:** Исключительно числовые столбцы выбросов конкретных газов: `air_solid_emissions`, `air_so_emissions`, `air_no_emissions`, `air_co_emissions`.

* **Суть задачи:** С помощью алгоритма `KMeans` или `DBSCAN` разделить города России на экологические кластеры без использования учительской разметки. Студенты должны интерпретировать результаты (например, выявить кластеры «металлургических гигантов» с упором на SO₂ или «угольных регионов»).

* **Доступные модели в MLABC:** `KMeans`, `DBSCAN`.

* **Рекомендация:** Перед кластеризацией обязательно применить `StandardScaler`. Экспериментировать с количеством кластеров $K$ (от 3 до 6). Вывести средние значения выбросов по каждому кластеру для интерпретации.

### 5. План обработки и анализа данных для студентов

Чтобы успешно обучить модели, студентам ФИИТ предстоит пройти через полноценный пайплайн предобработки данных, спотыкаясь о реальные инженерные «подводные камни».

#### Шаг 1. Загрузка и первичный анализ (EDA)

Загрузите данные с помощью `DataFrame.FromCsv`:

```pascal
uses MLABC;

begin
  var df := DataFrame.FromCsv('data_air_cities_raw.csv');
  df.Head(10).Print;
  df.Schema.Print;
  df.Describe.Print;
  df.MissingCounts.Print;
end.
```

Обратите внимание на:
- Пропуски: `MissingCounts` покажет количество NA по каждому столбцу.
- Типы столбцов: `Schema.Print` покажет, какие столбцы распознаны как числовые, а какие — как строковые.
- Распределения: числовые столбцы выбросов имеют тяжелые хвосты (аномальные значения в крупных промышленных центрах).

#### Шаг 2. Очистка скрытых и явных пропусков (Data Cleaning)

**Числовые фильтры:** В колонках выбросов (`air_solid_emissions` — `air_co_emissions`) пропущенные данные на заводах заменены техническим маркером `-1.0`. Прямое скармливание этих данных модели разрушит обучение.

Алгоритм очистки:
1. Заменить `-1.0` на `NA` с помощью `Filter` или `WithColumnFloat`.
2. Применить `Imputer` со стратегией `ImputeStrategy.isMedian` (медиана устойчива к выбросам).

```pascal
// Замена -1.0 на NA через фильтрацию
var df := DataFrame.FromCsv('data_air_cities_raw.csv');
var cols := |'air_solid_emissions', 'air_so_emissions', 
             'air_no_emissions', 'air_co_emissions'|;

// Заполнение пропусков медианой
var imputer := new Imputer(ImputeStrategy.isMedian);
df := imputer.FitTransform(df);
```

**Строковые заглушки:** Значения `No substance` и `No information` необходимо перевести в явный пустой класс или закодировать как отсутствие негативных факторов.

**Фильтрация таргета:** Строки, где пропущен итоговый класс `air_general_level` (473 наблюдения), должны быть удалены из обучающей выборки для классификации, так как для них нет учительской разметки.

```pascal
df := df.Drop('air_general_level');  // удалить строки с NA в target
```

#### Шаг 3. Главный методический капкан: Коррекция изменений СанПиН

Это самая глубокая аналитическая задача в проекте. Студенты должны обнаружить (или учесть по подсказке из ТЗ), что стандарты расчета уровня загрязнения менялись дважды:

1. **В 2014 году** Роспотребнадзор резко увеличил (ослабил) ПДК формальдегида более чем в три раза. Из-за этого на графиках и в таблице комплексный индекс ИЗА и уровень `air_general_level` во многих городах формально упали (воздух якобы «стал чище»), хотя физические объемы выбросов не снижались. Если учитывать прежние ПДК, количество городов с превышениями в 2014 году составило бы 199 вместо 174.

2. **В 2021 году** были введены жесткие нормативы СанПиН 1.2.3685-21, которые вернули строгие оценки, из-за чего показатели загрязнения на бумаге снова резко поползли вверх.

**Решение для ML:** Студенты не могут просто перемешать все года в случайном порядке. Им необходимо:

* Ввести синтетический категориальный признак `sanpin_era` с помощью `WithColumnInt` или `WithColumnString`:

```pascal
var df := DataFrame.FromCsv('data_air_cities_raw.csv');
// Добавление эры СанПиН
df := df.WithColumnInt('sanpin_era', row -> 
  begin
    var y := row.ToTuple['year'];
    if y < 2014 then Result := 0
    else if y < 2021 then Result := 1
    else Result := 2;
  end);
```

* Либо использовать ручное разбиение выборки по годам: обучать на данных до 2014 + 2014–2020, тестировать на 2021+.

#### Шаг 4. Кодирование категориальных признаков

**Географические признаки:** `region` и `city` — высокомощные категориальные признаки (сотни уникальных значений). `OneHotEncoder` создаст слишком много разреженных колонок.

Рекомендуемые стратегии:
- Использовать `OrdinalEncoder` для кодирования регионов и городов.
- Сгруппировать данные по макрорегионам, используя числовые коды `region_oktmo` (первые 2 цифры кода ОКТМО определяют федеральный округ).

```pascal
// Кодирование региона через OrdinalEncoder
var encoder := new OrdinalEncoder();
var encodedDf := encoder.FitTransform(df, 'region');
```

**Текстовые списки веществ:** Колонки `air_standard_index` и `air_qcp` содержат списки веществ (например: `бенз(а)пирен, пыль, формальдегид`). Расшифровка сокращений приведена в п. 3 (спецификация признаков).

Поскольку в MLABC нет `MultiLabelBinarizer`, студентам нужно:
1. Извлечь уникальные вещества из всех строк (вручную или через вспомогательную функцию).
2. Создать бинарные столбцы для каждого вещества через `WithColumnInt` (1 — вещество присутствует, 0 — отсутствует).

```pascal
// Пример: бинарный признак для формальдегида
df := df.WithColumnInt('has_formaldehyde', row ->
  begin
    var idx := row.ToTuple['air_qcp'].ToString.IndexOf('Ф');
    Result := if idx >= 0 then 1 else 0;
  end);
```

#### Шаг 5. Масштабирование признаков

Перед подачей в модели все числовые признаки необходимо отмасштабировать:

```pascal
var scaler := new StandardScaler();
var X_scaled := scaler.FitTransform(X);
```

**Важно:** Масштабирование критично для `KNNRegressor`, `KNNClassifier`, `KMeans` и `DBSCAN`, так как эти алгоритмы используют расстояния между точками.

#### Шаг 6. Разделение на обучающую и тестовую выборки

Используйте `TrainTestSplit` с фиксированным `seed` для воспроизводимости:

```pascal
var (X_train, X_test, y_train, y_test) := 
  Validation.TrainTestSplit(X, y, 0.2, 42);
```

**Внимание:** Из-за изменений СанПиН (Шаг 3) рекомендуется также проверить качество модели при разделении по годам, а не случайном перемешивании.

### 6. Известные ограничения данных

1. **Полнота данных:** Во многих населенных пунктах действует одна или две станции мониторинга (`air_stantions`). Такое количество станций не позволяет проводить полноценный мониторинг качества атмосферного воздуха на всей территории города. Исследование КБ «Стрелка» по данным спутникового мониторинга продемонстрировало иные оценки качества воздуха в крупных и средних городах.

2. **Сопоставимость данных:** Нормативы СанПиН предельно допустимых концентраций для отдельных веществ существенно менялись в 2014 и 2021 годах (см. Шаг 3). Это создаёт временной структурный сдвиг, который необходимо учитывать при обучении моделей.

3. **Несбалансированность целевой переменной:** Классы `air_general_level` распределены неравномерно. Класс `4` (очень высокий) встречается значительно реже остальных, что требует взвешенных метрик при классификации.
