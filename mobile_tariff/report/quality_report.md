# Отчёт о качестве датасета Mobile Tariff

## Размер
- 100 строк, 6 признаков + целевая переменная
- Задача: регрессия

## Типы признаков
| Признак | Тип | Категория |
|---------|-----|-----------|
| internet_gb | Float | Числовой |
| voice_minutes | Int | Числовой |
| sms_count | Int | Числовой |
| roaming_flag | Bool | Бинарный |
| rollover_flag | Bool | Бинарный |
| messenger_unlimited_flag | Bool | Бинарный |
| price_rub | Float | Целевая переменная |

## Пропуски и выбросы
- Пропусков нет
- Выбросов нет (данные синтетические)

## Категориальные признаки
- Категориальные признаки отсутствуют.

## Бинарные признаки
- `roaming_flag`: 20% тарифов
- `rollover_flag`: 32%
- `messenger_unlimited_flag`: 48%

## Целевая переменная
- Мин: ~141 руб., макс: ~840 руб., среднее: ~355 руб.

## Рекомендованная обработка
1. true/false → 0/1
2. Масштабирование числовых признаков
3. Разбиение 80/20

## Методология
- Модель: KNNRegressor (K=3)
- Предобработка: StandardScaler
- Метрики: R², MAE, RMSE

## Результаты
| Метрика | Значение |
|---------|----------|
| R² | 0.71 |
| MAE | 65.68 руб. |
| RMSE | 82.40 руб. |

## Вывод
Модель не идеальна — можно улучшать (менять K, добавлять признаки). Это хорошо для учебных целей: студент видит, что есть куда расти.
