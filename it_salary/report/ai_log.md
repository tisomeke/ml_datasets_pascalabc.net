# Лог использования ИИ — it_salary

**Дата:** 9 июля 2026 г.

## Инструменты

| Этап | Инструмент | Цель |
|------|-----------|------|
| Сбор референсных данных | Gemini Deep Research (web) | Исследование рынка IT-вакансий РФ: зарплаты, распределения, корреляции |
| Проектирование и генерация | Claude Code (Opus 4.8) | Проектирование структуры, генерация синтетических данных, написание кода и отчётов |
| Подготовка отчётной документации | Yandex SourceCraft (Roo Code) | Верификация отчётов, уточнение статистик, подготовка quality_report, README, sources.md, ai_log.md |

## Цели использования

1. **Gemini Deep Research** — сбор статистики по зарплатам, грейдам, специализациям, городам, форматам работы. Результат: `AGENT_WORK/Анализ ИТ-Рынка Труда РФ.md`
2. **Claude Code (Agentic)** — проектирование признаков и формулы зависимости, написание Python-скрипта генерации, создание 3 CSV-файлов (easy/medium/raw), meta-файлов, примеров программ на PascalABC.NET, шаблонов отчётной документации.
3. **Yandex SourceCraft (Roo Code)** — верификация отчётной документации: выверка статистик (пропуски, диапазоны зарплат, список городов), уточнение типов признаков, подготовка финальных версий quality_report.md, README.md, sources.md и данного лога.

## Ручная проверка

| Проверка | Результат |
|----------|-----------|
| Соответствие CSV требованиям (UTF-8, snake_case, латиница, запятая, точка) | ✅ |
| Соответствие meta-файлов содержимому CSV (имена, типы, количество столбцов) | ✅ |
| Корректность типов признаков (числовые, категориальные, DateTime, string) | ✅ |
| Корректность целевой переменной (salary_rub, регрессия) | ✅ |
| Пропуски обозначены как NA | ✅ |
| 3 версии: easy (без пропусков), medium (~7%), raw (~15% + выбросы) | ✅ |
| Размер 750 строк (в диапазоне 500–1000) | ✅ |
| Отсутствие персональных данных | ✅ |
| Разумность значений зарплат (соответствие рыночным диапазонам) | ✅ |
| Примеры программ компилируются и запускаются | ✅ |
| Статистики в quality_report соответствуют данным | ✅ |
